# PRD - 大学生智能化学习辅助工具 (StudyHub)

## 1. 项目概览

### 核心目标
本项目旨在为大学生提供一个轻量、高效、实时的**智能化学习辅助工具（StudyHub）**。结合生成式 AI 的 Vibe Coding 模式，利用现代化 Serverless 架构快速构建，帮助学生管理每日学习任务、记录课堂笔记，并通过社区功能进行互助交流。

## 2. 功能详细定义

整个系统围绕“学习辅助”展开，分为认证、任务管理、共享社区三大核心模块：

### 2.1 用户认证模块 (Supabase Auth)
* **邮箱注册与登录**：支持学生使用邮箱和密码进行安全的账号注册与登录。
* **登录状态保持**：使用 JWT 进行状态管理，用户刷新页面无需重新登录。
* **匿名/OFF测试模式**：在开发阶段关闭邮箱强制验证（Confirm Email = OFF），方便助教和老师评审测试。

### 2.2 任务管理模块 (个人学习助手)
* **今日待办 (Todo List)**：支持用户添加、标记完成、删除学习任务。
* **倒计时/番茄钟**：设置核心考试科目（如：操作系统、外语）的倒计时。
* **数据隔离**：利用 Supabase RLS（行级安全策略），确保学生只能查看和操作自己的任务数据。

### 2.3 共享笔记与互助社区 (公共留言板)
* **全部笔记/留言**：展示所有同学分享的学习心得或资料链接。
* **我的笔记/留言**：快速筛选出自己发布的内容。
* **实时同步**：基于 PostgreSQL 数据库，最新发布的内容会即时刷新在列表最前端（按 `created_at` 降序排列）。
* **字数限制**：单条留言或心得最多支持 200 字，确保排版整洁。

## 3. UX Flow (用户体验流程)

### 3.1 访问与认证流程
1.  **未登录状态**：访问首页 $\rightarrow$ 拦截并重定向至 `Sign In / Up` 页面 $\rightarrow$ 输入邮箱密码。
2.  **注册/登录成功**：跳转至主工作台（Dashboard）。

### 3.2 任务与社区交互流程
1.  **查看数据**：主页默认加载“全部留言/共享笔记”列表。
2.  **发布心得**：点击输入框 $\rightarrow$ 输入文本（若超过 200 字则限制输入） $\rightarrow$ 点击“提交/등록” $\rightarrow$ 列表无刷新实时更新。
3.  **筛选状态**：点击“全部留言”切换至公共流；点击“我的留言”切换至仅看自己，非本人内容提供删除按钮。

## 4. 技术栈与架构设计

本项目采用前后端分离但同构的现代化 Serverless 全栈架构：

* **前端框架 (Frontend)**: `Next.js` (App Router) - 提供极佳的响应式 UI 体验，完美适配手机与 PC 浏览器。
* **代码托管与版本管理**: `GitHub` - 至少保持 3 次以上的有效 Commit 提交记录。
* **后端与数据库 (Backend & DB)**: `Supabase`
    * **Auth**: 托管用户登录与权限安全。
    * **PostgreSQL**: 存储用户任务与社区留言数据。
    * **RLS (Row Level Security)**: 保护个人数据的隐私安全。
* **部署平台 (Deployment)**: `Vercel` - 绑定 GitHub 仓库，实现代码 Push 后的自动构建、加密环境变量集成与全球 CDN 加速。

## 5. 数据库设计 (SQL 脚本示例)

请在 Supabase 的 **SQL Editor** 中运行以下脚本，以创建所需的表和安全策略：

```sql
-- 创建学习辅助工具留言/笔记表
create table public.study_notes (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  user_id uuid references auth.users not null,
  user_email text not null,
  content text not null check (char_length(content) <= 200)
);

-- 启用行级安全控制 (RLS)
alter table public.study_notes enable row level security;

-- 允许所有已认证用户查看笔记
create policy "Allow authenticated users to read notes" 
on public.study_notes for select 
using (auth.role() = 'authenticated');

-- 允许已认证用户插入自己的笔记
create policy "Allow authenticated users to insert their own notes" 
on public.study_notes for insert 
with check (auth.uid() = user_id);

-- 允许用户删除自己的笔记
create policy "Allow users to delete their own notes" 
on public.study_notes for delete 
using (auth.uid() = user_id);
